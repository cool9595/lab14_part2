## Ответы на вопросы

**1.** Во сколько раз параллельное перемножение матриц быстрее последовательного? Совпадает ли ускорение с количеством ядер?

> Ответ: Для маленькой матрицы 3x3 ускорения нет (0.00x) — overhead создания 9 процессов больше вычислений. Для больших матриц ускорение ~2-4x, но не равно числу ядер из-за накладных расходов ОС

**2.** Как изменяется время выполнения при увеличении числа процессов в Pool?

> Ответ: Pool(1) = последовательное. Pool(2) быстрее. Pool(4) ещё быстрее. Дальше рост процессов не даёт ускорения — лимит ядер CPU + overhead

**3.** Почему для передачи результатов из процессов нельзя использовать обычные глобальные переменные?

> Ответ: Каждый процесс ОС имеет **собственное адресное пространство**. Изменения в одном процессе не видны в другом. Нужны IPC: Queue, Pipe, SharedMemory

**4.** Почему асинхронная версия в `01_sync_vs_async.py` выполняется быстрее синхронной, хотя использует только одно ядро?

> Ответ: Во время `await asyncio.sleep()` управление передаётся event loop, который выполняет другие корутины. Синхронный `time.sleep()` блокирует весь поток

**5.** В каком порядке выводятся сообщения при запуске нескольких клиентов через `asyncio.gather()`?

> Ответ: Порядок **не детерминирован** — зависит от планировщика event loop и времени выполнения каждой корутины. Может быть 1-2-3, 2-1-3, 3-1-2 и т.д

**6.** Что произойдёт, если в асинхронном сервере использовать `time.sleep()` вместо `await asyncio.sleep()`?

> Ответ: time.sleep() **заблокирует весь event loop** — сервер перестанет обрабатывать новые подключения, пока спит. await asyncio.sleep() отпускает управление

**7.** Сравните PID в многопроцессном сервере и многопоточном сервере из lab 2. Почему в multiprocessing PID разные, а в threading — одинаковые?

> Ответ: multiprocessing: каждый клиент = отдельный процесс ОС → разные PID. threading: все потоки в одном процессе => одинаковый PID. Потоки делят память процесса.
